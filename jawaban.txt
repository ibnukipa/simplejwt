> Jelaskan bagaimana anda menggunakan GIT untuk proses development sampai deployment production
	Berdasarkan yang saya pernah lakukan, pastinya menggunakan beberapa branch pada GIT sesuai dengan kebuthan, untuk branch master biasanya dipegang oleh PM, sedangangkan untuk branch develompment tergantung pada metode atau cara pengerjaan project. kalau untuk branch production biasa hanya untuk menyesuaikan deplyment ke server production dan harus sudah melalui proses verifikasi terlebih dahulu baik PM maupun pihak yang bersangkutan.
	Perlu untuk dicatat bahwasanya setiap programmer yang menggunakan git dan tim, harus code clean maksudnya adalah code yang di push adalah code hanya yang benar-benar dijalankan, untuk komen diperlukan jika logik atau proses membutuhkan pengetahuan ekstra untuk memahaminya.
> Apa yang anda ketahui dengan code refactoring? Jika tahu, bagaimana anda melakukannya?
	Sepengetahuan saya, code refactoring itu adalah merubah secara struktural aplikasi (minim perubahan logik), untuk proses melakukan dibuat terlebih dahulu rancangan yang akan diterapkan pada aplikasi target (kemungkinan aplikasi target ini memiliki rancangan yang kurang baik maka perlu dilakukan nya code refactoring)
> Apa yang anda ketahui dari technical debt? Jika tahu bagaimana anda mengatasinya?
	Sebenarnya belum pernah dengar. tetapi mungkin saya pernah mengalaminya.
> Bagaimana anda mengukur kualitas suatu aplikasi?
	Sebenanya ini adalah judul tugas akhir saya, jadi untuk mengukur kualitas sebuah aplikasi sudah terdapat standar pengukuran (internasional, ex: ISO/IEC 25023). Pada proses pengukuran kualitas aplikasi terdapat banyak sekali aspek yang dapat dinilai. Salah satu contoh yang perlu diukur adalah aspek pemeliharaan dan kegunaan. yang dimana aspek pemeliharaan memliki banyak poin yang perlu dinilai (ex: reusabale class, coupling between class, dan cyclometic compelxity) sedangkan untuk aspek penggunaan fokus pada fitur-fitur aplikasi yang bertujuan untuk apakah aplikasi adalah aplikasi yang sudah menyelesaikan masalah atau sudah memenuhi kebutuhan pengguna (aspek ini dipengaruhi salah satunya oleh elisitasi kebutuhan).
